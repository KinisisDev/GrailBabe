## 2. Page Layout

### Header
- Title: "Trading board"
- Subtitle: "Browse and post item trades with other collectors"
- Top-right: "Post a trade" button (opens a Post Trade modal — see section 4)

### Stats bar (3 metric cards in a row)
- Active listings: 248
- New today: 34
- Completed trades: 91
(hardcoded numbers for now)

### Toolbar
Row with:
- Filter tabs: All | TCG | LEGO | Sports Cards | My Watchlist
- Search input (filters cards by have/want item name as user types)
- Sort dropdown: Newest first | Most recent activity | Near me
- Distance dropdown (only visible when "Near me" sort is selected): Within 25 miles | 50 miles | 100 miles | Any distance

### Trade card grid
Responsive grid — 3 columns on desktop, 2 on tablet, 1 on mobile.

---

## 3. Trade Card Component

Each card shows:

**Top row:** category badge (colored by type) + status badge (Open = green, Pending = amber) + time posted

**Trade items row:** two item boxes side by side with a → arrow between them

- Left box (HAVE): label "HAVE", icon placeholder, item name, detail (set · grade)
- Right box (WANT): dashed border style, label "WANT"
  - If openToOffers is false: show item name + detail
  - If openToOffers is true: show "Open to offers" in italic + detail as subtext

**Footer row:** user avatar (colored circle with initials) + username on left, "Make offer" button on right
- If status is "pending": show "Pending" text instead of the button, button is disabled/muted
- "Make offer" button navigates to /messages (use wouter's useLocation or Link)

Category badge colors:
- tcg → blue tint
- lego → green tint  
- sports → amber tint

Avatar colors map: blue, green, amber, purple, coral, teal — use bg and text CSS variables or subtle colored backgrounds consistent with the app theme.

---

## 4. Post a Trade Modal

Clicking "Post a trade" opens a modal (React state, no backend):

Fields:
- Category: dropdown — TCG | LEGO | Sports Cards
- Sub-category: text input (e.g. "Pokémon", "Star Wars")
- Have — Item name: text input
- Have — Condition/Grade: text input (e.g. PSA 9, Near Mint, Sealed)
- Want — Open to offers: toggle checkbox
- Want — Item name: text input (disabled/hidden if open to offers is checked)
- Want — Condition/Grade: text input (disabled/hidden if open to offers is checked)
- Your zip code: text input (5-digit)

Submit button: "Post trade" — adds the new listing to the top of the local mock array using React state and closes the modal.

Modal close button (×) in top-right corner.

---

## 5. Filtering & Sorting Logic (all client-side React state)

- Active filter tab filters trades by category (all shows everything)
- Search input filters by have.name or want.name (case-insensitive includes)
- "My Watchlist" tab: show only trades where category is "tcg" (placeholder behavior)
- Sort "Newest first": default order
- Sort "Most recent activity": reverse order (mock)
- Sort "Near me": use the zipcodes npm package to calculate distance between the logged-in user's zip (hardcode "90210" as the current user's zip for now) and each listing's zip. Sort ascending by distance. Apply the distance dropdown filter to hide listings beyond the selected radius. Show "~X miles away" below the username in the card footer when "Near me" is active.

Install zipcodes package if not already present: npm install zipcodes
