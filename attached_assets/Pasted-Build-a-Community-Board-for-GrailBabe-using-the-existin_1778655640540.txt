## 1. Drizzle Schema

Add to the existing schema file (wherever posts/vault items are defined):

```ts
// Posts
export const posts = pgTable("posts", {
  id: uuid("id").primaryKey().defaultRandom(),
  authorId: uuid("author_id").notNull().references(() => users.id, { onDelete: "cascade" }),
  category: text("category").notNull(), // 'tcg' | 'lego' | 'sports' | 'grail' | 'price-check' | 'general'
  title: text("title").notNull(),
  body: text("body").notNull(),
  isPinned: boolean("is_pinned").default(false),
  createdAt: timestamp("created_at").defaultNow(),
});

// Comments
export const postComments = pgTable("post_comments", {
  id: uuid("id").primaryKey().defaultRandom(),
  postId: uuid("post_id").notNull().references(() => posts.id, { onDelete: "cascade" }),
  authorId: uuid("author_id").notNull().references(() => users.id, { onDelete: "cascade" }),
  body: text("body").notNull(),
  createdAt: timestamp("created_at").defaultNow(),
});

// Votes (one row per user per post)
export const postVotes = pgTable("post_votes", {
  userId: uuid("user_id").notNull().references(() => users.id, { onDelete: "cascade" }),
  postId: uuid("post_id").notNull().references(() => posts.id, { onDelete: "cascade" }),
  value: integer("value").notNull(), // 1 or -1
}, (t) => ({
  pk: primaryKey({ columns: [t.userId, t.postId] }),
}));
```

Run the Drizzle migration after adding the schema.

## 2. API Endpoints

Add these endpoints following the same pattern as the existing routes in the app:

**Posts:**
- GET /api/posts — list posts, query params: category (optional), sort ('hot'|'new'|'top', default 'hot')
  - Each post returns: id, category, title, body, isPinned, createdAt, authorId, authorName, score (sum of votes), commentCount
  - Pinned posts always first, then sorted by selected sort
  - 'hot' = score desc, 'new' = createdAt desc, 'top' = score desc all time
- POST /api/posts — create a post (auth required), body: { category, title, body }
- GET /api/posts/:id — single post with full body + author info + score + commentCount

**Comments:**
- GET /api/posts/:id/comments — all comments for a post, ordered by createdAt asc, include authorName
- POST /api/posts/:id/comments — add a comment (auth required), body: { body }

**Votes:**
- POST /api/posts/:id/vote — upsert a vote (auth required), body: { value: 1 | -1 }
  - If the user already voted the same value, remove the vote (toggle off)
  - If the user voted the opposite value, update to new value

## 4. Page 1 — artifacts/web/src/pages/Community.tsx

### Layout: two-column (left sidebar + main feed)

**Left sidebar (~190px):**
- Section label "Categories"
- Nav items: All Posts, TCG, LEGO, Sports Cards, Grail Posts, Price Checks, General
- Each shows a colored dot and the count of posts in that category
- Clicking filters the feed
- Section label "My Activity"
- Nav items: My Posts, My Upvoted (placeholders for now, no functionality needed)

**Main feed:**

Header row: "Community" title + "New post" button (opens modal)

Compose bar: avatar circle with user initials + "Share something with the community..." placeholder text (clicking opens new post modal)

Sort tabs: Hot | New | Top (updates the sort param passed to useListPosts)

Search input: client-side filter on post titles as user types

**Post list (not a grid — full width stacked cards):**

Each post card:
- Left vote column: upvote chevron → score number → downvote chevron
  - Clicking upvote calls useVotePost with value 1
  - Clicking the same vote again removes it (toggle)
  - Downvote uses value -1
  - Active vote highlighted in neon blue (--primary)
- Right content area:
  - Top: category badge (color-coded per category) + "Pinned" label if isPinned
  - Title (font-weight 500)
  - Body snippet truncated to 1 line
  - Meta: author avatar + name, comment count, time ago
- Pinned posts always appear at top of list
- Clicking the card navigates to /community/:id
- Hover state: border brightens slightly

**New Post Modal (React state):**
- Category dropdown: TCG | LEGO | Sports Cards | Grail Post | Price Check | General
- Title input (required, max 200 chars, show char count)
- Body textarea (required, min 20 chars)
- Submit "Post to community" → calls useCreatePost, closes modal, refreshes list
- Cancel / × button closes modal

**Empty state** when no posts in selected category:
Centered icon + "No posts yet. Be the first to post in this category."
