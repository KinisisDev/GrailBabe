## 1. Drizzle Schema

Add/update the profiles table:

```ts
export const profiles = pgTable("profiles", {
  id: uuid("id").primaryKey().references(() => users.id, { onDelete: "cascade" }),
  screenname: text("screenname").unique().notNull(),
  displayName: text("display_name"),
  bio: text("bio"),
  city: text("city"),
  region: text("region"),
  postalCode: text("postal_code"),
  country: text("country").default("US"),
  lat: real("lat"),
  lng: real("lng"),
  onboardingComplete: boolean("onboarding_complete").default(false),
  screennameChangedAt: timestamp("screenname_changed_at").defaultNow(),
  formerScreenname: text("former_screenname"),
  formerScreennameChangedAt: timestamp("former_screenname_changed_at"),
  createdAt: timestamp("created_at").defaultNow(),
});
```

## 2. API Endpoints

**GET /api/profiles/check-screenname?screenname=xyz**
Returns { available: true/false }
Real-time availability check during onboarding.

**POST /api/profiles**
Create profile on onboarding (auth required).
Body: { screenname, displayName, city, region, postalCode, country }
- Validate screenname: 3–20 chars, alphanumeric + underscores only, unique
- Geocode city+region+country via OpenStreetMap Nominatim (free, no API key):
  fetch(`https://nominatim.openstreetmap.org/search?city=CITY&country=COUNTRY&format=json`)
  Store first result's lat/lng
- Set onboardingComplete = true on the profile row
- // TODO: also call Clerk API to set publicMetadata.onboardingComplete = true when Clerk is activated

**GET /api/profiles/me**
Current user's own full profile (auth required).

**GET /api/profiles/:screenname**
Public profile by screenname.
Returns: screenname, displayName, bio, city, region, postalCode, country,
memberSinceYear (year from profile.createdAt),
formerScreenname (only if formerScreennameChangedAt within last 30 days),
vaultItemCount, estimatedVaultValue, tradesCompleted,
tradeRepScore, communityPostCount,
featuredVaultItems (up to 3), recentPosts (up to 5),
grailList (public items only), recentTradeReviews (up to 3)

**PATCH /api/profiles/me**
Update profile fields (auth required).
Body: any fields except screenname.
Re-geocode if location fields change.

**PATCH /api/profiles/me/screenname**
Change screenname (auth required).
Body: { screenname }
- If screennameChangedAt < 120 days ago → return 400 with daysRemaining
- Store old screenname → formerScreenname, set formerScreennameChangedAt = now
- Update screenname + screennameChangedAt = now

## 4. Onboarding Guard

Create artifacts/web/src/components/OnboardingGuard.tsx

Use the same auth pattern already used in Messages and Community 
to get the current user. Check if their profile row has 
onboardingComplete = true. If not, redirect to /onboarding.

```tsx
// TODO: when Clerk is activated, also check 
// user.publicMetadata.onboardingComplete here

function OnboardingGuard({ children }) {
  const { data: profile, isLoading } = useGetMyProfile()
  const [, setLocation] = useLocation()

  if (isLoading) return <LoadingSpinner />
  if (profile && !profile.onboardingComplete) {
    setLocation("/onboarding")
    return null
  }
  return children
}
```

Wrap the main app layout (AppShell) with OnboardingGuard in App.tsx.
The /onboarding route must sit OUTSIDE the guard.
